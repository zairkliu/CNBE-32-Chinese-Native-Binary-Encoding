# Linux 0.01 CNBE-32 RISC-V

A technical exploration: porting **Linux 0.01** (the first Linux kernel by Linus Torvalds, 1991) from x86 to **RISC-V 64** with **CNBE-32 (Chinese Native Binary Encoding)** — creating a fully Chinese-native operating system kernel.

> ⚠️ **Disclaimer: This project was auto-generated by an AI Agent (Codex/GPT-5). It has NOT been tested on RISC-V hardware or QEMU and does NOT compile in its current state. This is a proof-of-concept and technical exploration only.** See [WHITEPAPER.md](WHITEPAPER.md) for the full whitepaper.

## Overview

This project translates Linux 0.01 into a Chinese-native OS with four major components:

| Component | Description |
|-----------|-------------|
| **CNBE-32 Runtime** | Unicode↔CNBE mapping, radical/stroke/structure extraction, semantic distance |
| **Chinese BASIC Interpreter** | 16 Chinese keywords, Chinese variable names, program mode, user functions |
| **Chinese Compiler** | Lexer → recursive descent parser → bytecode → stack VM (27 instructions) |
| **Chinese Shell** | Full interactive command-line interface in Chinese |

### Target Hardware

| Parameter | Spec |
|-----------|------|
| CPU | RISC-V 1GHz (QEMU virt platform) |
| RAM | 1GB |
| L3 Cache | 32MB (can hold 81.6KB CNBE table) |
| Storage | 1GB |

## Known Issues

The code has several known blocking issues. See the full list in [WHITEPAPER.md](WHITEPAPER.md#已知问题). Key problems include: encoding corruption in some files, Sv32/Sv39 page table mismatch, unimplemented trap handler functions, and incomplete user mode switching.
