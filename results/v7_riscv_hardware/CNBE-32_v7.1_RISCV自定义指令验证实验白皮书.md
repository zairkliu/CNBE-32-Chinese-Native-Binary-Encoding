## 三、指令集设计

### 3.1 cnhe.map — Unicode → CNBE 查表

```
cnhe.map rd, rs1
// rd  = skill_table[rs1 - 0x4E00]
// rs1 = Unicode 码点 (0x4E00-0x9FA5)
// 未找到时 rd = 0

Encoding: 0000000 | rs1(5) | 00000 | 000 | rd(5) | 0001011
// custom-0 opcode: 0001011
```

### 3.2 cnhe.extract — CNBE 位域提取

```
cnhe.extract rd, rs1, field
// field=00: rd = (rs1 >> 24) & 0xFF  (部首区)
// field=01: rd = (rs1 >> 19) & 0x1F  (笔画数)
// field=10: rd = (rs1 >> 15) & 0xF   (结构区)
// field=11: rd = (rs1 >> 4)  & 0x7FF (字库索引)

Encoding: 0000000 | rs1(5) | field(3) | 001 | rd(5) | 0001011
```

### 3.3 cnhe.cmp — CNBE 语义相似度比较

```
cnhe.cmp rd, rs1, rs2
// rd = hamming_weight(rs1 ^ rs2)
// 加权汉明距离，用于语义相似度度量

Encoding: 0000000 | rs2(5) | rs1(5) | 010 | rd(5) | 0001011
```

## 四、性能分析

### 4.1 指令计数分析

`cnhe_lookup` 函数的 RISC-V 汇编指令分解：

| 操作 | 指令数 | 说明 |
|:----|:------:|:-----|
| 边界检查 `u ≥ 0x4E00` | 2 | lui + bltu |
| 边界检查 `u ≤ 0x9FA5` | 3 | lui + addiw + bltu |
| 数组索引计算 | 3 | addi + slli + add |
| 内存加载 | 1 | lw (L1 命中) |
| 函数调用/返回 | 2 | jal + ret |
| **合计（函数调用实现）** | **~22** | **含 2 条分支指令** |
| **cnhe.map 硬件实现** | **1** | **单条指令，无分支** |

### 4.2 不同实现的性能对比

| 实现方式 | 指令数 | 周期数 | 相对加速 |
|:---------|:------:|:------:|:--------:|
| 函数调用（当前 C）| ~22 | ~27 | 1.0x |
| 内联展开 | ~15 | ~18 | 1.5x |
| **cnhe.map 硬件指令** | **1** | **2-5** | **5-13x** |
