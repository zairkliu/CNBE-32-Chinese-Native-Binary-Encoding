## 四、技术文档

### 4.1 API 参考

```c
/**
 * cnhe_lookup - 模拟 RISC-V cnhe.map 指令
 * @unicode: 汉字 Unicode 码点 (0x4E00-0x9FA5)
 * 返回: 32-bit CNBE 编码，0 表示未找到
 *
 * RISC-V 指令映射: cnhe.map rd, rs1
 * rs1 = unicode 码点
 * rd  = 32-bit CNBE 编码
 */
uint32_t cnhe_lookup(uint32_t unicode);

/**
 * cnhe_decode - 模拟 RISC-V cnhe.extract 指令
 * @code:  32-bit CNBE 编码
 * @radical:   输出-部首区 (bits 31:24)
 * @stroke:    输出-笔画数 (bits 23:19)
 * @structure: 输出-结构区 (bits 18:15)
 * @idx:       输出-字库索引 (bits 14:4)
 *
 * RISC-V 指令映射: cnhe.extract rd, rs1, field
 */
void cnhe_decode(uint32_t code, uint32_t *radical, uint32_t *stroke,
                 uint32_t *structure, uint32_t *idx);
```

### 4.2 CNBE 32-bit 编码位域

```
 1 0 9 8 7 6 5 4 3 2 1 0 9 8 7 6 5 4 3 2 1 0 9 8 7 6 5 4 3 2 1 0
+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+
|     部首区(8)  |  笔画(5)  | 结构(4) |     字库区(11)      |扩展(4)|
+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+-+
```

### 4.4 存储需求

| 存储层级 | 容量 | 可容纳 Skill 表 |
|:---------|:----:|:---------------:|
| L1 缓存 | 32-64 KB | ❌ 仅部分 |
| L2 缓存 | 256-512 KB | ✅ **完整容纳** |
| L3 缓存 | 2-32 MB | ✅ 完整容纳 |
| 片上 ROM | 128 KB+ | ✅ 完整容纳 |
| 外部 Flash | 1 MB+ | ✅ 完整容纳 |

### 4.5 向量化批量查表

```c
/* SIMD/向量化批量查表 - 一次处理 8 个字符 */
 * RISC-V V 扩展映射：vcnhe.map vd, vs1
 * 向量查表可映射到 RISC-V 向量扩展（V扩展），
 * 实现单指令多数据（SIMD）操作，进一步提升吞吐量。*/
void cnhe_lookup_batch(const uint32_t *unicode_in,
                       uint32_t *code_out, int count) {
    for (int i = 0; i < count; i++) {
        code_out[i] = cnhe_lookup(unicode_in[i]);
    }
}
```
