# 编码架构与设计哲学

## 核心理念

CNBE-32 不是为显示文本而设计的编码，而是为**机器理解中文**而设计的编码。

### 三原则

1. **原生性**: 编码空间直接对应认知维度（部首/笔画/结构），CPU直接提取
2. **可计算性**: 编码字段支持位运算、比较、汉明距离（`cnhe.cmp` 指令）
3. **跨领域通用性**: 结构化位域思想可应用于非语言领域

### 编码即先验

CNBE的核心假设：**编码的先验知识是最廉价的算力**。
在 v9.0-v10.8 的 9 个跨领域验证中，CNBE 在零训练/零微调条件下：

| 领域 | 实验 | CNBE表现 | 环境 |
|------|------|----------|------|
| 语言学 | v2/v6.5 | +17~+81pp | qwen3.5/Gemma4B |
| 生态学 | v9.0 | 86%优于Raw | PyTorch |
| 气象学 | v10.3 | -19%误差 | Python纯实现 |
| 金融学 | v10.0-10.2 | 双市场正收益 | PyTorch+sklearn |
| 生物学 | v10.4 | 差3.6pp | Python纯实现 |
| 物理学 | v10.5 | R2 0.60-0.77 | sklearn |
| 社会学 | v10.6 | 优于Random | sklearn |
| 预训练底座 | v10.7 | 初始损失差0.09 | PyTorch Transformer |
| 数学推理 | v10.8 | 全面优于OneHot | PyTorch Transformer |

## 与现有体系的关系

| 体系 | 定位 | CNBE关系 |
|------|------|----------|
| Unicode | 全球字符标识 | CNBE是Unicode之上的语义增强层 |
| GB 18030 | 国标字符集 | CNBE为其语义扩展 |
| Unicode IDS | 结构描述文本 | CNBE将描述转为可运算枚举 |
| 仓颉/五笔 | 输入法编码 | CNBE汲取结构拆解思想 |

## 硬件路径

v7.0-v7.2 完成 RISC-V 自定义指令验证：
- `cnhe.map`: Unicode→CNBE查表（1周期）
- `cnhe.extract`: 位域提取（1周期）
- `cnhe.cmp`: 加权汉明距离（1-2周期）
