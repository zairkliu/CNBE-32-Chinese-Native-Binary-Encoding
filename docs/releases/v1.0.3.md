# CNBE-32 v1.0.3 Release Notes

Status: released (2026-07-10)

This document prepares release notes for `v1.0.3`.

These checks passed before the v1.0.3 release on 2026-07-10.

---

## Summary

`v1.0.3` is an SDK hardening checkpoint for CNBE-32.

It focuses on stricter public API validation, deterministic database batch lookup behavior, and stronger release artifact checks.

---

## Highlights

- Hardened CNBE-32 decode input validation.
- Hardened direct `CNBE32(...)` construction validation.
- Preserved input order and duplicate characters in database batch lookups.
- Removed a UTF-8 BOM from a packaged Python module.
- Expanded format integrity coverage to include the packaged experimental encoder module.
- Strengthened release artifact verification for exact artifact counts and versioned filenames.
- Preserved Python, C, and Rust golden vector consistency checks.

---

## Stable scope

The stable baseline covers:

- CNBE-32 bitfield encode/decode.
- Strict field validation.
- Strict encoded-value validation.
- Bit-level Hamming distance utility.
- Field-weighted distance compatibility utility.
- Basic CJK database lookup when the database is available.
- Ordered database batch lookup with duplicate preservation.
- Golden vector consistency checks.

---

## Experimental scope

The following remain experimental:

- Extended CJK coverage toward the 97,686-character research target.
- Hardware-oriented prototypes.
- RISC-V experiments.
- LLM / JEPA experiments.
- OS / kernel prototypes.
- Cross-domain research experiments.

---

## Non-goals

This release does not claim:

- full CJK coverage,
- zero-collision rate,
- production readiness,
- validated model performance,
- validated hardware correctness,
- or validated performance on 90,000+ characters.
