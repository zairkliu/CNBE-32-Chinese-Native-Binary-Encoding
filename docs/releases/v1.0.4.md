# CNBE-32 v1.0.4 Release Notes

This document prepares release notes for `v1.0.4`.

`v1.0.4` is a standards-restart runtime data checkpoint for CNBE-32. It focuses
on the 8105 national-standard core, the reproducible Agent workflow, and the
controlled transition away from legacy AI-generated catalog fields.

## Highlights

- Keeps the 20,902-row packaged Python SDK runtime scope.
- Treats the 8105 common standardized Chinese character table as the core
  release-track baseline.
- Applies a conservative standardized runtime repair to rows already present in
  the runtime model.
- Rebuilds both packaged SQLite databases from the updated runtime source.
- Adds explicit history boundaries for older AI-generated test data.
- Keeps tag, GitHub release, and PyPI publication as separate manual gates.

## Non-Goals

This release candidate does not:

- claim 97,686-row validation;
- turn ZDIC/cache data into national-standard evidence;
- promote outside-8105 rows as national-standard output;
- insert the 276 runtime-missing 8105 rows without an index policy;
- publish a GitHub release;
- publish to PyPI.
