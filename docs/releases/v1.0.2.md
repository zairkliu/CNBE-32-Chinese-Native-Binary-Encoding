# CNBE-32 v1.0.2 Release Notes Draft

Status: draft

This document prepares release notes for `v1.0.2`.

No tag, GitHub Release, or PyPI publication is created by this PR.

---

## Summary

`v1.0.2` is a release-readiness checkpoint for CNBE-32.

It focuses on repository integrity, release verification, and implementation consistency across Python, C, and Rust.

---

## Highlights

- Restored repository format integrity across key Python, TOML, YAML, Markdown, C, Makefile, and Rust files.
- Added a repository-wide format integrity validation script.
- Preserved Python SDK build, wheel install, pytest, and ruff checks.
- Preserved Python / C / Rust golden vector consistency checks.
- Added release artifact verification for wheel and source distribution contents.
- Clarified release process requirements before any future tag or PyPI publication.

---

## Stable scope

The stable baseline covers:

- CNBE-32 bitfield encode/decode.
- Strict field validation.
- Bit-level Hamming distance utility.
- Field-weighted distance compatibility utility.
- Basic CJK database lookup when the database is available.
- Golden vector consistency checks.

---

## Experimental scope

The following remain experimental:

- Extended CJK coverage toward the 97,686-character research target.
- Hardware-oriented prototypes.
- RISC-V experiments.
- LLM / JEPA experiments.
- OS / kernel prototypes.
- Cross-domain research experiments.

---

## Verification checklist

Before tagging `v1.0.2`, run:

```bash
python scripts/validate_format_integrity.py
python scripts/verify_release_artifacts.py
python -m compileall src tests
python -m build
python -m pip install --force-reinstall dist/*.whl
python -c "import cnbe32; print(cnbe32.__all__)"
pytest
ruff check src tests
make -C c/golden_vectors clean
make -C c/golden_vectors test
cargo test --manifest-path rust/golden_vectors/Cargo.toml
```

---

## Non-goals

This release does not claim:

- full CJK coverage,
- zero-collision rate,
- production readiness,
- validated model performance,
- validated hardware correctness,
- or validated performance on 90,000+ characters.
