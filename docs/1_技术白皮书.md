**版本**: 3.0 | **日期**: 2026-07-02  
**覆盖**: 《通用规范汉字表》8,105 字 + CJK 全部统一汉字 97,686 字

---

## 一、背景与问题

汉字在数字时代始终面临"编码寄宿"困境——从 ASCII 到 Unicode，汉字始终被编码为外来体系的附属物。UTF-8 中一个汉字 3 字节（英文仅 1 字节），且编码本身不携带任何汉字结构信息。

## 二、CNBE-32 编码方案

32 位定长编码，将汉字字形语义直接映射到二进制位域：

| 位域 | 宽度 | 取值 | 描述 |
|------|:----:|:----:|------|
| [31:24] 部首区 | 8bit | 1-214 | 康熙部首编号 |
| [23:19] 笔画区 | 5bit | 1-31 | 总笔画数 |
| [18:15] 结构区 | 4bit | 0-11 | 9 种结构类型 |
| [14:4] 字库区 | 11bit | 0-2047 | 组内索引 |
| [3:0] 扩展区 | 4bit | 0 | 繁简/多音标志 |

**核心优势**: ① 语义内嵌 ② 定长 O(1) 随机访问 ③ 编码值即字典序 ④ 位域可直接被 CPU 指令提取

## 三、技术实现

### RISC-V 自定义指令 (6 条)

| 指令 | 功能 | 周期 |
|:----:|:----:|:----:|
| cnbe.enc | Unicode → CNBE-32 | 2 (哈希) |
| cnbe.dec | CNBE-32 → Unicode | ~4053 (线性) |
| cnbe.rad | 提取部首 | **1** |
| cnbe.str | 提取笔画 | **1** |
| cnbe.struct | 提取结构 | **1** |
| cnbe.dist | 语义距离 | **2** |
