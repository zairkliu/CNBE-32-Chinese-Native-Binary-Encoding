# Hanzi Standard Learning Plan

This plan is generated by `scripts/build_hanzi_standard_learning.py`.

The goal is to learn national language and writing standards before any CNBE-32 encoding audit or mapping work.
PDF originals are recorded by filename only; converted source/OCR assets under `cnbe-research/source` and `cnbe-research/knowledge` are the working inputs.

## Gates

- `encoding_generation_gate`: `NO_GO`
- `sqlite_build_gate`: `NO_GO`
- Next allowed action: human reading review, schema design, and small pilot evidence rows only.

## Stop Rules

- Stop if a formal field is supported only by OCR.
- Stop if a term has no primary standard source.
- Stop if a structure name is outside the twelve allowed structures or `独体字`.
- Stop if a script attempts to write CNBE mapping values.
- Stop if a large PDF, OCR batch, dictionary, image slice, or Wikipedia payload would be committed.

## Learning Acceptance Checklist

- Every one of the 13 Hanzi knowledge items is present in the term model.
- Every term has at least one primary standard source or a documented methodological source.
- Every national standard is represented by filename only and mapped to converted JSON/Markdown assets.
- Every converted source has identity metadata in the machine-readable audit report.
- The structure vocabulary is closed over the twelve allowed structures plus `独体字`.
- OCR-only evidence is marked as review-needed rather than accepted as authority.
- Dictionary, etymology, and Wikipedia sources remain context or adjudication evidence only.
- Current CNBE values remain diagnostic and cannot be used as self-evidence.
- Encoding generation remains blocked until a human review packet is designed and approved.

## Next Human Reading Pass

A human or reviewer agent should now read the generated term report and structure report, compare them against the
converted standard assets, and mark any disputed definitions before schema extraction starts. The first permitted
implementation step after that review is a small evidence-row pilot, not a full database and not a code table.
